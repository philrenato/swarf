3. CHIP PHYSICS — web/kiri/swarf-chips.js (NEW)
   Tool-spin-aware particle layer. Four flavors share InstancedMesh pools:
   - helical curl (55% — long ductile-cut ribbon, TubeGeometry around a
     CatmullRom helix)
   - triangular flake (18% — brittle)
   - tight C-coil (20% — aluminum short curl)
   - grit cube (7% — MDF / HDPE powder)
   Spawn gated by "cutting motion": XY motion present AND tool z near/below
   stock top. Per-spawn physics: random point on cutter perimeter, velocity
   = spin-tangent + feed-direction kick + upward pop, gravity + drag each
   RAF. On z≤0 chips flatten to a random horizontal rotation and write to a
   settled-chip ring buffer (cap 1600; oldest fall off). Airborne cap 240.
   IMPORT clears everything.
   Plumbing: new `swarf.tool.move` event emitted from
   src/kiri/mode/cam/app/anim-2d.js:273 and anim-3d.js:308 for negative-id
   meshes (tool only). swarf-chips.js polls for window.moto.space + kiri.api
   before attaching.

4. PHASE AURA — web/kiri/swarf-phase.js (NEW) + CSS in swarf.css
   Body class swarf-phase-{idle,toolpaths,previewing,simulating,exported}
   reflects the workflow step. CSS drives a gentle filter change per phase;
   post-EXPORT is grayscale(0.92) contrast(0.94) brightness(0.90) — the
   scene goes quiet until the user clicks/keys/wheels, at which point
